## SLIDE 2: The Imperative

**Why IAM Transformation Now?**

| Past State | Future State |
|------------|--------------|
| ❌ Manual provisioning (2+ days) | ✅ HR-driven, real-time (under 1 hour) |
| ❌ Standing privileged accounts | ✅ Just-in-time, zero standing |
| ❌ 40% SSO coverage | ✅ 90%+ SSO + passwordless |
| ❌ Cloud IAM = Shadow IT | ✅ Cloud governance + CIEM |
| ❌ IAM = Compliance checkbox | ✅ IAM = Competitive advantage |

**The Cost of Inaction:**
- Security risk: Orphaned accounts, overprivileged users
- Operational drag: 40% of IT tickets = password resets
- User friction: 3.2/5 satisfaction score
- Audit exposure: Hours to produce evidence
